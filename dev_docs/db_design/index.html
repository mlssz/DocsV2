1.1 repositories

1.1.1 locations Object

key key in Chinese type default description
id 位置id Number(int) unique, required >=1, 该值用于定位,顺时针算, “17” id为1
place 块号 Number(int) required >=1 该值仅用于显示，根据文档图，分四个大块，该值表示第几块（顺时针算）

available_height 剩余高度 Number 2 该位置剩余的高度。
is_top_full 顶层是否满了 Boolean false 最顶层是否满
materials 存放的物资 Array<ObjectId> []  

1.2 materials

key key in Chinese type default description
id 物资编号 Number now >=1, utc时间数字
type 物资类型 Number 0 哪些类型未定
description 物资描述 String ""  
import_time 入库时间 Date now  
estimated_export_time 估计出库时间 Date   用于自动搬移将出库的物资到门口
height 物资高度 Number 0  
length 物资长度 Number 0  
width 物资宽度 Number 0  
repository_id 储存仓库 Number required 所在仓库的id, -1值意味着出库; 搬运到新地方，修改的是这两个字段
location_id 仓库中位置 Number requried  
status 物资状态 Number 300 状态码，详细见下方
migrations 物资搬运记录 Array<ObjectId> [] 这个应当是一个队列，以创建时间从晚（距今更近）到早（距今更远）的顺序排序
location_update_time 位置更新时间 Date now 这个是系统中位置更新时间，实际更新时间应当直接记录在migrations中

1.2.1 status

  • 1 开头为位置正确
    • 100 ：物资处于正确位置中
    • 101 : 物资已出库
  • 2 开头为处于搬运中
    • 200 ：物资信息录入完毕，正在搬运入库
    • 201 ：物资从原位置搬运到一个新的位置
    • 202 ：物资正在搬运出库
  • 3 开头为位置不正确
    • 300 ：物资信息录入完毕，未入库
    • 301 : 物资位置在系统中已变换，但是未搬运到新的位置
    • 302 ：物资在系统中已申请出库，但是未搬运出库
    • 303 ：物资位置错误
  • 4 开头为物资缺失
    • 400 : 物资找不到（一般为识别满损坏）

1.4 tasks

key key in Chinese type default description
action 动作 Number 500 动作码详细见下方
staff 接受任务的员工 ObjectId    
status 任务状态 Number 0 0未开始1进行中2完成3任务取消
migration 移动信息 ObjectId   当action为6开头时，该值为空
err_repository 错误仓库 Number   当action为5开头时，该值为空
err_location 错误位置 Number   当action为5开头时，该值为空
publish_time 任务发布时间 Date now  
start_time 任务开始时间 Date    
end_time 任务结束时间 Date    
remark 任务附加评语 String   一般用于任务取消时

1.4.1 action

  • 5 开头为移动
    • 500 ：移动入库
    • 501 ：从原位置移动到新位置
    • 502 ：移动出库
  • 6 开头为异常处理
    • 600 ：校正物资错误位置
    • 601 ：解决物资摆放方式异常或识别码损坏而造成系统无法识别的问题
